### Insights from the new vs old comparison

1. **Most reactions agree at the 30–50 kJ/mol level.** 61% of the 20,570 shared reactions are within 50 kJ/mol, and 75% are within 100 kJ/mol. For directionality and pathway-feasibility analysis, these are usable. The median absolute difference of 32 kJ/mol is meaningful but not catastrophic for typical metabolic-engineering use cases.

2. **The new model honestly reports much wider uncertainty.** The old model's median posterior std of 1.5 kJ/mol is misleadingly tight — that's the formal posterior of a tightly-determined small linear system, not the empirical error on real reactions. The new model's wider posterior (median ≈84 kJ/mol) is a more honest reflection of the underlying group-contribution method's extrapolation risk on novel chemistry.

3. **The expanded feature space is the main source of disagreement.** The new model has 39,230 features vs 26,404 in the original, but is trained on the same 4,001 reactions. Many of the 12,826 *new* radius-2 groups appear in only 1–2 training reactions, so their coefficients are weakly determined. When these rare groups appear in test reactions, predictions can shift substantially. This is the trade-off for covering more compounds.

4. **The 4.3% of reactions with \|Δ\| > 500 kJ/mol are outliers, not the norm.** They're dominated by macromolecule biosynthesis reactions (polysaccharides, lipid chains, peptides) where compound group counts lie far outside the training distribution. Both models are extrapolating wildly here; they just diverge differently. Filter these out (`abs_delta_dG < 500` or even `< 100`) for any quantitative downstream analysis.

5. **Pearson correlation of 0.16 looks alarming but is dominated by outliers.** The distribution is heavy-tailed: trim to \|delta_dG\| < 100 (75% of reactions) and the correlation rises substantially. The raw correlation should not be interpreted as "the models disagree everywhere" — it's pulled down by a small number of macromolecule extrapolations with thousand-kJ deltas.

6. **Best practice for using these predictions:** filter to reactions where **both** `old.dG_std` and `new.dG_std` are small (e.g. < 50 kJ/mol), or where the two predictions agree to within their combined uncertainty. Use the new model for the 11,354 reactions outside the old reference set, and treat extreme magnitudes as flags rather than usable values.

## How It Was Generated

```bash
# 1. Retrain the model on all ModelSEED structures (one-time, ~75 min)
.venv/bin/python retrain_modelseed.py --modelseed_dir ../ModelSEEDDatabase --skip_predict

# 2. Bulk-predict all feasible reactions (~13 min)
.venv/bin/python predict_all_modelseed.py
```

## Key Implementation Notes

- **Why a separate batch script?** The naive per-reaction loop made 31,924 separate calls to `BayesianRidge.predict(..., return_std=True)`, each performing an O(p²) covariance computation on the 39,230-dimensional feature space. The batch script (`predict_all_modelseed.py`) builds the full (31,924 × 39,230) feature matrix once and calls `predict()` in chunks of 2,000, dropping wall-time from "infeasible" to ~13 minutes.

- **Per-compound vector caching:** Compound→fingerprint conversion is done lazily and cached. Featurization of all 31,924 reactions takes ~7 seconds on a single core.

- **pH/ionic-strength correction:** Computed via the same Legendre transform as the original dGPredictor (`Compound.transform_pH7`), using ModelSEED's stored pKa values rather than running ChemAxon at prediction time.

## Customizing the Predictions

To regenerate with different physiological conditions:

```bash
# Different pH
.venv/bin/python predict_all_modelseed.py --pH 6.5 \
    --output data/modelseed_dG_pH6.5.json

# Different ionic strength
.venv/bin/python predict_all_modelseed.py --ionic_strength 0.15 \
    --output data/modelseed_dG_I0.15.json
```
